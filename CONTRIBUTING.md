# Contributing Guide | Guia de Contribuicao

## Portugues

Obrigado por contribuir com esta organizacao.

Esta organizacao GitHub e mantida como uma comunidade academica aberta para estudantes do **UNIPE (Centro Universitario de Joao Pessoa)** e outros colaboradores que queiram ajudar a construir recursos educacionais acessiveis e projetos open source liderados por estudantes.

### Formas de Contribuir

Voce pode contribuir de varias maneiras, incluindo:

- melhorar documentacao
- corrigir erros ou links quebrados
- enviar guias de estudo, anotacoes ou exercicios
- propor ideias para novos repositorios
- contribuir com codigo em projetos open source
- revisar pull requests e reportar problemas

Estudantes sao especialmente incentivados a contribuir com materiais de estudo que possam ajudar outras pessoas a aprender melhor.

### Antes de Comecar

Por favor:

- leia o README do repositorio
- verifique se ja existem issues ou pull requests relacionados
- siga a estrutura e o estilo de formatacao do repositorio
- mantenha as contribuicoes claras, corretas e faceis de revisar

### Como Fazer Fork de um Repositorio

1. Abra o repositorio para o qual deseja contribuir.
2. Clique em **Fork** no canto superior direito do GitHub.
3. Crie sua propria copia no seu perfil do GitHub.
4. Clone seu fork para a sua maquina local:

```bash
git clone https://github.com/your-username/repository-name.git
```

5. Crie uma nova branch para suas alteracoes:

```bash
git checkout -b my-change
```

### Fazendo Alteracoes

Ao preparar uma contribuicao:

- mantenha as alteracoes focadas e relevantes
- use nomes de arquivos claros e markdown limpo
- verifique as informacoes antes de enviar materiais de estudo
- respeite direitos autorais e nao envie conteudo que voce nao pode compartilhar
- evite incluir informacoes privadas ou sensiveis

### Abrindo um Pull Request

1. Envie sua branch para o seu fork:

```bash
git push origin my-change
```

2. Abra um pull request para o repositorio original.
3. Use o template de pull request e explique suas alteracoes com clareza.
4. Relacione issues relevantes quando necessario.

### Regras Basicas de Contribuicao

- Seja respeitoso e colaborativo.
- Mantenha o conteudo profissional e util.
- Escreva em portugues ou ingles de forma clara, seguindo o padrao definido pelo repositorio.
- Certifique-se de que sua contribuicao faz sentido para o objetivo do repositorio.
- Esteja aberto a feedback e ajustes solicitados.

### Contribuicoes de Materiais de Estudo

Se voce estiver contribuindo com conteudo academico, garanta que:

- o material esteja organizado e legivel
- o conteudo seja de sua autoria ou possa ser compartilhado legalmente
- as informacoes estejam corretas dentro do seu melhor conhecimento
- o material ajude outros estudantes de forma pratica

Exemplos incluem:

- roadmaps de estudo
- listas de exercicios
- resumos
- guias de revisao
- projetos de exemplo

### Precisa de Ajuda?

Se voce nao souber por onde comecar, abra uma issue e descreva o que gostaria de contribuir. Mantenedores ou membros da comunidade podem ajudar a orientar.

Toda melhoria util importa, incluindo pequenas correcoes e primeiras contribuicoes.

## English

Thank you for contributing to this organization.

This GitHub organization is maintained as an open academic community for students of **UNIPE (Centro Universitario de Joao Pessoa)** and other contributors who want to help build accessible educational resources and student-led open source projects.

### Ways To Contribute

You can contribute in many ways, including:

- improving documentation
- fixing errors or broken links
- submitting study guides, notes, or exercises
- proposing new repository ideas
- contributing code to open source projects
- reviewing pull requests and reporting issues

Students are especially encouraged to contribute study materials that can help others learn more effectively.

### Before You Start

Please:

- read the repository README
- check for existing issues or pull requests
- follow the repository structure and formatting style
- keep contributions clear, accurate, and easy to review

### How To Fork a Repository

1. Open the repository you want to contribute to.
2. Click **Fork** in the top-right corner of GitHub.
3. Create your own copy under your GitHub account.
4. Clone your fork to your local machine:

```bash
git clone https://github.com/your-username/repository-name.git
```

5. Create a new branch for your changes:

```bash
git checkout -b my-change
```

### Making Changes

When preparing a contribution:

- keep changes focused and relevant
- use clear file names and clean markdown
- verify information before submitting study materials
- respect copyright and do not upload content you do not have permission to share
- avoid including private or sensitive information

### Opening a Pull Request

1. Push your branch to your fork:

```bash
git push origin my-change
```

2. Open a pull request against the original repository.
3. Use the pull request template and explain your changes clearly.
4. Link any related issues when applicable.

### Basic Contribution Rules

- Be respectful and collaborative.
- Keep content professional and helpful.
- Write in Portuguese or English clearly, following the repository standard.
- Make sure your contribution fits the purpose of the repository.
- Be open to feedback and requested changes.

### Study Material Contributions

If you are contributing academic content, please make sure that:

- the material is organized and readable
- the content is your own work or can be shared legally
- the information is correct to the best of your knowledge
- the material helps other students learn in a practical way

Examples include:

- study roadmaps
- exercise lists
- summaries
- revision guides
- sample projects

### Need Help?

If you are unsure where to start, open an issue and describe what you would like to contribute. Maintainers or community members can help guide you.

Every useful improvement matters, including small corrections and first-time contributions.
